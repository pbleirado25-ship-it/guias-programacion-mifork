# TEMA 1. Clases y objetos

## 1. ¿Cuáles son las cuatro características básicas de la programación orientada a objetos? Describe brevemente cada una

### Respuesta

Las cuatro características básicas de la programación orientada a objetos son la **abstracción**, la **encapsulación**, la **herencia** y el **polimorfismo**. Estas propiedades permiten modelar problemas complejos de forma más organizada y cercana a la realidad.

La **abstracción** consiste en representar solo los aspectos relevantes de un objeto, ignorando los detalles internos. La **encapsulación** agrupa datos y operaciones en una misma entidad y permite controlar el acceso a dichos datos.

La **herencia** posibilita crear clases nuevas a partir de otras existentes, reutilizando código y ampliando funcionalidades. El **polimorfismo** permite que una misma operación se comporte de forma diferente dependiendo del objeto que la ejecute.

## 2. Cita cuatro lenguajes populares que permitan la programación orientada a objetos

### Respuesta

Existen muchos lenguajes que soportan la programación orientada a objetos. Uno de los más representativos es **Java**, diseñado desde su origen con este paradigma como base principal.

Otros lenguajes ampliamente utilizados son **C++**, que añade orientación a objetos al lenguaje C, **Python**, que combina orientación a objetos con otros paradigmas, y **C#**, muy empleado en el ecosistema .NET y con un modelo similar al de Java.

## 3. Los paradigmas anteriores a la POO, ¿Qué es la **programación estructurada**? y, todavía mejor, ¿Qué es la **programación modular**?

### Respuesta

La **programación estructurada** es un paradigma que se basa en el uso de estructuras de control bien definidas, como secuencias, condiciones y bucles. Su objetivo principal es evitar saltos incontrolados en el flujo del programa y mejorar la claridad del código.

La **programación modular** propone dividir un programa en módulos independientes, cada uno con una responsabilidad concreta. En lenguajes como C, estos módulos suelen corresponder a ficheros fuente y cabeceras.

Ambos enfoques mejoran la organización del código, pero no integran de forma natural los datos con las funciones que los manipulan, algo que sí aporta la programación orientada a objetos.

---

## 4. ¿Qué tres elementos definen a un objeto en programación orientada a objetos?

### Respuesta

Un objeto en programación orientada a objetos se define por su **identidad**, su **estado** y su **comportamiento**. Estos tres elementos permiten describir completamente a un objeto dentro de un sistema.

La **identidad** distingue a un objeto de cualquier otro, incluso si comparten los mismos valores. El **estado** está formado por los valores actuales de sus atributos. El **comportamiento** corresponde a los métodos que definen las acciones que puede realizar.

## 5. ¿Qué es una clase? ¿Es lo mismo que un objeto? ¿Qué es una instancia? ¿Todos los lenguajes orientados a objetos manejan el concepto de clase?

### Respuesta

Una **clase** es una definición o plantilla que describe cómo serán los objetos: qué datos contendrán y qué operaciones podrán realizar. No ocupa memoria por sí misma para los datos de los objetos.

Un **objeto** es una **instancia** concreta de una clase creada en tiempo de ejecución. Por tanto, clase y objeto no son lo mismo: la clase define, mientras que el objeto existe y almacena datos reales.

No todos los lenguajes orientados a objetos utilizan clases de la misma manera. Algunos lenguajes, como los basados en prototipos, prescinden del concepto clásico de clase, aunque siguen trabajando con objetos.

---

## 6. ¿Dónde se almacenan en memoria los objetos? ¿Es igual en todos los lenguajes? ¿Qué es la **recolección de basura**?

### Respuesta

En lenguajes orientados a objetos como Java, los objetos se almacenan normalmente en el **montón (heap)** de memoria. Las variables que los referencian suelen almacenarse en la pila o en estructuras estáticas.

El lugar exacto y la forma de almacenamiento pueden variar entre lenguajes. En C++, por ejemplo, un objeto puede almacenarse tanto en la pila como en el montón, dependiendo de cómo se cree.

La **recolección de basura** es un mecanismo automático que libera la memoria ocupada por objetos que ya no son accesibles. En Java, este proceso lo gestiona la máquina virtual, evitando que el programador libere memoria manualmente.

## 7. ¿Qué es un método? ¿Qué es la **sobrecarga de métodos**?

### Respuesta

Un **método** es una función asociada a una clase u objeto, que define una operación que puede realizarse sobre sus datos. A diferencia de las funciones en C, los métodos están ligados a objetos concretos.

La **sobrecarga de métodos** consiste en definir varios métodos con el mismo nombre pero con distintos parámetros. El lenguaje decide cuál ejecutar según el número o tipo de argumentos proporcionados.

Este mecanismo permite reutilizar nombres de operaciones para acciones conceptualmente similares, mejorando la legibilidad del código.

## 8. Ejemplo mínimo de clase en Java, que se llame Punto, con dos atributos, x e y, con un método que se llame `calculaDistanciaAOrigen`, que calcule la distancia a la posición 0,0. Por sencillez, los atributos deben tener visibilidad por defecto. Crea además un ejemplo de uso con una instancia y uso del método

### Respuesta

Una clase en Java se define usando la palabra clave `class`. Los atributos representan el estado del objeto y los métodos definen su comportamiento. En este ejemplo, la clase representa un punto en un plano cartesiano.

El método calcula la distancia al origen utilizando la fórmula matemática correspondiente. El uso de la clase implica crear una instancia y llamar al método sobre ella.

```java
class Punto {
    double x;
    double y;

    double calculaDistanciaAOrigen() {
        return Math.sqrt(x * x + y * y);
    }
}

class Prueba {
    public static void main(String[] args) {
        Punto p = new Punto();
        p.x = 3;
        p.y = 4;
        System.out.println(p.calculaDistanciaAOrigen());
    }
}
```

## 9. ¿Cuál es el punto de entrada en un programa en Java? ¿Qué es `static` y para qué vale? ¿Sólo se emplea para ese método `main`? ¿Para qué se combina con `final`?

### Respuesta

El punto de entrada de un programa en Java es el método `main`. Este método es el primero que ejecuta la máquina virtual al iniciar un programa.

La palabra clave `static` indica que un método o atributo pertenece a la clase y no a una instancia concreta. Por ello, puede usarse sin crear objetos. No se utiliza únicamente en `main`, sino también en métodos auxiliares o constantes.

Cuando se combina `static` con `final`, se define un valor constante compartido por toda la clase, similar a una constante global en C.

## 10. Intenta ejecutar un poco de Java de forma básica, con los comandos `javac` y `java`. ¿Cómo podemos compilar el programa y ejecutarlo desde linea de comandos? ¿Java es compilado? ¿Qué es la **máquina virtual**? ¿Qué es el *byte-code* y los ficheros `.class`?

### Respuesta

Para compilar un programa Java desde línea de comandos se utiliza `javac`, que genera uno o varios ficheros `.class`. Para ejecutar el programa se emplea el comando `java`, indicando la clase principal.

Java es un lenguaje **compilado e interpretado**. El código fuente se compila a *byte-code*, que no es código máquina específico de un sistema concreto.

La **máquina virtual de Java** es un entorno que ejecuta el *byte-code*. Los ficheros `.class` contienen este *byte-code*, permitiendo que el mismo programa funcione en distintas plataformas.

## 11. En el código anterior de la clase `Punto` ¿Qué es `new`? ¿Qué es un **constructor**? Pon un ejemplo de constructor en una clase `Empleado` que tenga DNI, nombre y apellidos

### Respuesta

La palabra clave `new` se utiliza para crear un objeto en memoria a partir de una clase. Este proceso reserva espacio y devuelve una referencia al nuevo objeto.

Un **constructor** es un método especial que se ejecuta al crear una instancia. Su nombre coincide con el de la clase y se utiliza para inicializar los atributos.

```java
class Empleado {
    String dni;
    String nombre;
    String apellidos;

    Empleado(String dni, String nombre, String apellidos) {
        this.dni = dni;
        this.nombre = nombre;
        this.apellidos = apellidos;
    }
}
```

## 12. ¿Qué es la referencia `this`? ¿Se llama igual en todos los lenguajes? Pon un ejemplo del uso de `this` en la clase `Punto`

### Respuesta

La referencia `this` apunta al objeto actual sobre el que se está ejecutando un método. Permite diferenciar entre atributos del objeto y variables locales.

No todos los lenguajes utilizan el mismo nombre para esta referencia, aunque el concepto es equivalente. En C++ también se usa `this`.

```java
class Punto {
    double x;
    double y;

    void setX(double x) {
        this.x = x;
    }
}
```

## 13. Añade ahora otro nuevo método que se llame `distanciaA`, que reciba un `Punto` como parámetro y calcule la distancia entre `this` y el punto proporcionado

### Respuesta

Este método permite calcular la distancia entre dos objetos de la misma clase. El objeto actual se referencia mediante `this`, mientras que el otro se recibe como parámetro.

La fórmula utilizada es la distancia euclídea entre dos puntos del plano.

```java
double distanciaA(Punto otro) {
    double dx = this.x - otro.x;
    double dy = this.y - otro.y;
    return Math.sqrt(dx * dx + dy * dy);
}
```

## 14. El paso del `Punto` como parámetro a un método, es **por copia** o **por referencia**, es decir, si se cambia el valor de algún atributo del punto pasado como parámetro, dichos cambios afectan al objeto fuera del método? ¿Qué ocurre si en vez de un `Punto`, se recibiese un entero (`int`) y dicho entero se modificase dentro de la función?

### Respuesta

En Java, los objetos se pasan **por copia de la referencia**. Esto significa que el método recibe una copia de la referencia al objeto, pero ambos apuntan al mismo objeto en memoria.

Por ello, si se modifican los atributos del objeto dentro del método, los cambios son visibles fuera. Sin embargo, si se reasigna la referencia, el objeto original no se ve afectado.

En el caso de tipos primitivos como `int`, se pasa una copia del valor, por lo que cualquier modificación dentro del método no afecta a la variable original.

## 15. ¿Qué es el método `toString()` en Java? ¿Existe en otros lenguajes? Pon un ejemplo de `toString()` en la clase `Punto` en Java

### Respuesta

El método `toString()` se utiliza para obtener una representación en texto de un objeto. Es muy útil para depuración y para mostrar información al usuario.

Este concepto existe en otros lenguajes, aunque con distintos nombres o mecanismos. En Java, todas las clases heredan este método de la clase base `Object`.

```java
@Override
public String toString() {
    return "Punto(" + x + ", " + y + ")";
}
```

## 16. Reflexiona: ¿una clase es como un `struct` en C? ¿Qué le falta al `struct` para ser como una clase y las variables de ese tipo ser instancias?

### Respuesta

Una clase puede recordar a un `struct` en C, ya que ambos agrupan datos relacionados. Sin embargo, un `struct` solo contiene datos y no comportamiento asociado.

A una clase le falta al `struct` la posibilidad de definir métodos que operen directamente sobre los datos, así como mecanismos de encapsulación y control de acceso.

Además, las clases incorporan conceptos como constructores, herencia y polimorfismo, que no existen de forma nativa en C.

## 17. Quitemos un poco de magia a todo esto: ¿Como se podría “emular”, con `struct` en C, la clase `Punto`, con su función para calcular la distancia al origen? ¿Qué ha pasado con `this`?

### Respuesta

En C, se puede emular una clase utilizando un `struct` para los datos y funciones que reciban dicho `struct` como parámetro. La función actúa de forma similar a un método.

El papel de `this` lo desempeña explícitamente el puntero al `struct` que se pasa a la función.

```c
typedef struct {
    double x;
    double y;
} Punto;

double distanciaAOrigen(Punto* p) {
    return sqrt(p->x * p->x + p->y * p->y);
}
```

Aquí, el parámetro `p` cumple la función que en Java realiza implícitamente `this`.
